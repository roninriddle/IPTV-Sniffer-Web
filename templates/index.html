运行拓扑

        宿主机必须能看到机顶盒 IPTV 流量
      

      
0.6 将组播候选、FCC 快速换台字段、channelAcquire 令牌记录、截图预览和多格式导出集中在同一页面。容器只能抓取宿主机实际可见的流量。

      

        

          
推荐：管理型交换机镜像口

          
光猫 IPTV 口 → 交换机 A 口
交换机 B 口 → 机顶盒
交换机 C 口 → 抓包设备 / Docker 宿主机

推荐配置：
源端口：B 口（机顶盒端口）
目标端口：C 口（抓包设备）
镜像方向：Both / 双向

          
A 口也可以镜像到 C，但优先推荐镜像 B 口，便于观察机顶盒实际收到的 IPTV 流量。

        

        

          
无管理型交换机

          
光猫 IPTV 口
      ↓
OpenWrt / iStoreOS
      ↓
机顶盒

          
前提是 IPTV 流量真实经过这台设备，并在本页选择对应 IPTV 接口抓包。

          
不可用场景：光猫 IPTV 口直接接机顶盒，而 NAS / Docker 只在普通 LAN 中。
