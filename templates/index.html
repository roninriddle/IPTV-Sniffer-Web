使用说明

            
先确认拓扑，再嗅探整理，最后导出或定时更新 EPG。

            
容器只能抓到宿主机实际可见的 IPTV 流量。推荐使用交换机镜像机顶盒端口，或让 IPTV 流量真实经过运行 Docker 的设备。

          

          

            
            
            
          

        


        

          

            
              运行拓扑与抓包前提
              宿主机必须能看到机顶盒 IPTV 流量
            
            

              

                
推荐：管理型交换机镜像口

                
光猫 IPTV 口 → 交换机 A 口
交换机 B 口 → 机顶盒
交换机 C 口 → 抓包设备 / Docker 宿主机

源端口：B 口（机顶盒端口）
目标端口：C 口（抓包设备）
镜像方向：Both / 双向

              

              

                
OpenWrt / iStoreOS 串接

                
光猫 IPTV 口
      ↓
OpenWrt / iStoreOS
      ↓
机顶盒

                
不可用场景：光猫 IPTV 口直接接机顶盒，而 NAS / Docker 只在普通 LAN 中。

              

            

          

        


        

          

            1. 运营商频道发现（主入口）
            填写机顶盒 IP，点击「开始捕获」后重启机顶盒，系统自动从 STB 开机 TCP 流量中解析运营商 EPG 门户的完整频道表（频道名、组播地址、FCC 参数），一键「导入到频道列表」。
          

          

            2. 嗅探整理（补充入口）
            设置抓包网卡和 rtp2httpd 地址，切换机顶盒频道，系统过滤无效组播、自动识别编码/分辨率/截图和 EPG，点击「导入到频道列表」将嗅探结果并入频道列表。
          

          

            3. 频道列表与导出
            在「频道列表」中查看所有已导入的频道，可勾选特定频道后点击「生成播放列表」，一键输出直连 M3U、源地址 M3U、JSON、TXT、CSV 等格式。
          

          

            4. 定时 EPG 刷新
            在「定时 EPG」中管理 XMLTV EPG 来源，可手动或定时刷新全部来源，保持频道 EPG 与台标信息持续更新，匹配结果在导出时自动写入 tvg-id / tvg-logo 字段。
