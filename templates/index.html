使用说明

            
先完成运营商频道发现，再在线路页导出和维护 EPG。

            
容器通过交换机镜像口捕获机顶盒开机流量，从运营商频道表中发现频道、整理线路并提取认证信息。

          

          

            
            
            
            
          

        


        

          

            
              唯一推荐拓扑：交换机镜像口
              只捕获机顶盒开机流量
            
            

              

                
交换机镜像口

                
光猫 IPTV 口 → 交换机
  ├─ 机顶盒端口（被镜像源）
  └─ 宿主机端口（镜像目标）

                
✓ 抓机顶盒开机流量、频道表、FCC/FEC、Option60
✓ 生成频道线路和 rtp2httpd 源文件
✓ FNOS 容器需在「高级设置 → 功能」勾选 NET_RAW；实验性认证还需 NET_ADMIN
✗ 镜像口不能主动发 IGMP/FCC，因此不能代表 rtp2httpd 可直接播放

              

            

          

        


        

          

            1. 运营商频道发现（主入口）
            填写机顶盒 IP，点击「开始捕获」后重启机顶盒，系统自动从 STB 开机流量中解析频道表与 DHCP 认证信息（MAC / IPTV IP / Option60 等），一键「导入到频道线路」。
          

          

            2. 频道线路与导出
            确认频道名称、分组、主源和备线，按播放器或 rtp2httpd 需要导出最佳线路、全部线路或 HLS 播放列表。
          

          

            3. 播放诊断
            先检测 rtp2httpd、M3U、FCC、IGMP 和组播回流，判断问题在播放列表、配置、上游链路还是镜像口限制。
          

          

            4. IPTV 认证助手（高级）
            只有需要让本机主动播放时再使用；实验模式只操作选定接口，执行前备份，支持恢复。FNOS 需勾选 NET_ADMIN 和 NET_RAW。
          

          

            5. EPG 与台标维护
            在「频道线路」中管理 XMLTV EPG 与台标来源，可手动刷新并重新匹配 tvg-id / tvg-logo。

播放链路诊断

              按 rtp2httpd、认证、FCC、组播、M3U 分层定位播放失败点
            

            
点击「运行诊断」检查 rtp2httpd 是否可访问、FCC 是否响应、认证信息与频道列表是否就绪，给出排查结论。填写「检测频道」后，还会主动发起 IGMP 加入并检测是否收到 239.x UDP 组播流，并判断是否为镜像口（SPAN，仅能被动看到流量、无法主动播放）。组播链路检测需要宿主机 tcpdump 及 NET_ADMIN、NET_RAW 权限。

            

              镜像口说明：
              交换机镜像口只适合捕获机顶盒流量，不能代表本机能主动播放。若诊断显示“未加入也能看到流量、主动加入收不到”，请到「IPTV 认证」页按选定网口生成脚本或执行实验性认证，再让 rtp2httpd 使用完成认证的上游网口。
