使用说明

            
先确认拓扑，再嗅探整理，最后导出或定时更新 EPG。

            
容器只能抓到宿主机实际可见的 IPTV 流量。本向导只保留交换机镜像机顶盒端口这一种嗅探方式，用于发现频道、整理线路与提取认证信息。

          

          

            
            
            
            
          

        


        

          

            
              唯一推荐拓扑：交换机镜像口
              只嗅探，不主动拉流
            
            

              

                
交换机镜像口（只嗅探）

                
光猫 IPTV 口 → 交换机
  ├─ 机顶盒端口（被镜像源）
  └─ 宿主机端口（镜像目标）

                
✓ 抓机顶盒开机流量、频道表、FCC/FEC、Option60
✓ 生成频道线路和 rtp2httpd 源文件
✓ FNOS 容器需在「高级设置 → 功能」勾选 NET_RAW；实验性认证还需 NET_ADMIN
✗ 镜像口不能主动发 IGMP/FCC，因此不能代表 rtp2httpd 可直接播放

              

            

          

        


        

          

            1. 运营商频道发现（主入口）
            填写机顶盒 IP，点击「开始捕获」后重启机顶盒，系统自动从 STB 开机流量中解析频道表与 DHCP 认证信息（MAC / IPTV IP / Option60 等），一键「导入到频道线路」。
          

          

            2. IPTV 认证助手（需要主动播放时）
            使用已捕获的 MAC / Hostname / Option60 生成认证脚本；实验模式可只对选定接口执行认证，并在执行前备份初始网络状态，支持恢复。FNOS 中请到容器「高级设置 → 功能」勾选 NET_ADMIN 和 NET_RAW。
          

          

            3. 频道线路管理
            同名频道自动归组，按清晰度（4K → 1080p → 720p → SD）与探测状态自动选主源；也可手动设为主源。导出「最佳线路 M3U」（每组只取主源）或「全部线路 M3U」。
          

          

            4. 播放诊断
            检测 rtp2httpd 可达性、FCC 响应与认证状态，给出「M3U 问题 / rtp2httpd 配置问题 / 上游组播问题」结论，帮助定位播放失败原因。
          

          

            5. 定时 EPG 刷新
            在「定时 EPG」中管理 XMLTV EPG 来源，定时刷新保持 tvg-id / tvg-logo 持续更新。

播放链路诊断

              按 rtp2httpd、认证、FCC、组播、M3U 分层定位播放失败点
            

            
点击「运行诊断」检查 rtp2httpd 是否可访问、FCC 是否响应、认证信息与频道列表是否就绪，给出排查结论。填写「检测频道」后，还会主动发起 IGMP 加入并检测是否收到 239.x UDP 组播流，并判断是否为镜像口（SPAN，仅能被动嗅探、无法主动播放）。组播链路检测需要宿主机 tcpdump 及 NET_ADMIN、NET_RAW 权限。

            

              镜像口说明：
              交换机镜像口只适合嗅探机顶盒流量，不能代表本机能主动播放。若诊断显示“未加入也能看到流量、主动加入收不到”，请到「IPTV 认证」页按选定网口生成脚本或执行实验性认证，再让 rtp2httpd 使用完成认证的上游网口。
