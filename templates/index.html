使用说明

            
先确认拓扑，再嗅探整理，最后导出或定时更新 EPG。

            
容器只能抓到宿主机实际可见的 IPTV 流量。推荐使用交换机镜像机顶盒端口，或让 IPTV 流量真实经过运行 Docker 的设备。

          

          

            
            
          

        


        

          

            
              运行拓扑与抓包前提
              宿主机必须能看到机顶盒 IPTV 流量
            
            

              

                
推荐：管理型交换机镜像口

                
光猫 IPTV 口 → 交换机 A 口
交换机 B 口 → 机顶盒
交换机 C 口 → 抓包设备 / Docker 宿主机

源端口：B 口（机顶盒端口）
目标端口：C 口（抓包设备）
镜像方向：Both / 双向

              

              

                
OpenWrt / iStoreOS 串接

                
光猫 IPTV 口
      ↓
OpenWrt / iStoreOS
      ↓
机顶盒

                
不可用场景：光猫 IPTV 口直接接机顶盒，而 NAS / Docker 只在普通 LAN 中。

              

            

          

        


        

          

            1. 嗅探整理
            设置抓包网卡和 rtp2httpd 地址，切换机顶盒频道，系统会先过滤无效组播，再自动识别编码、分辨率、截图和 EPG。
          

          

            2. 人工补全
            自动识别失败时，只补频道名称和分类即可；表格会在编辑时暂停重绘，避免输入被打断。
          

          

            3. 定时 EPG
            定时任务只更新已经指定的 M3U 清单的 EPG 和台标，不再自动抓包，不会改动你的频道源地址。
